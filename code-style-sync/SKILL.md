- - # SKILL: Active Style-Sync & Evolution Protocol (自进化代码克隆协议)

    ## 🎯 核心目标
    本技能致力于通过“主动感知”和“知识归档”，消除 AI 与人类编码习惯之间的摩擦。它强制 Agent 在执行任务的同时，不断优化并扩充本地 `references/` 知识库。

    ## ⚙️ 核心执行 SOP (标准作业程序)

    ### Phase 1: 深度感知与特征提取 (Sensing)
    1. **多维扫描：** 接收任务后，优先扫描当前工作区。不仅要看文件名，更要理解代码逻辑：
       - 命名：是驼峰还是下划线？局部变量是否有特定前缀？
       - 结构：大括号位置、空格习惯、是否使用快读优化？
    2. **知识比对：** 将感知到的特征与 `references/` 目录下的 `strict_rules.md` 和 `algo_templates.cpp` 进行比对。
       - 若发现冲突，优先遵守用户在当前工作区展现的**最新习惯**，并记录差异。
  
    ### Phase 2: 任务执行与风格对齐 (Execution)
    1. **强制克隆：** 在生成新代码时，必须 100% 模仿 `references/` 中的模板神韵。
    2. **原理同步：** 每一段生成的核心代码上方，必须附带 1-2 行解释“为什么这样写更优”的原理注释，体现脚踏实地的底层思维。
  
    ### Phase 3: 自我维护与知识归档 (Evolution) - **关键步骤**
    1. **提炼新知：** 当检测到用户在当前项目中写出了比 `references/` 里更优雅、更高效的代码时，Agent 必须主动发起提议：
       > “*检测到你当前的 [函数名/模块] 实现了更优的复杂度/风格。是否需要我将其提取并更新至 references 知识库？*”
    2. **自动同步：** 获得授权后，Agent 需调用 `write_file` 工具，自动更新 `references/` 文件夹。
    3. **时间戳记录：** 每次更新后，在文件头部自动记录：`Update Log: [北京时间] - [更新内容概述]`。
  
    ## ⚠️ 严格约束 (Constraints)
    - **拒绝平庸：** 即使是模仿风格，也要寻找时间/空间复杂度最优解。
    - **禁止猜测：** 找不到参考时，必须询问用户“请指引一个参考文件路径”，而非随意生成。
    - **权限边界：** 严禁在未经用户确认的情况下直接覆盖 `references/` 中的核心规范。

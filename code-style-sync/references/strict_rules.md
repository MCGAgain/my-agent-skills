# 用户的硬性代码规范

在遵循本规范时，没有妥协的余地。

## 1. 命名规范
- **变量名/函数名：** 优先使用下划线命名法 (snake_case)，例如 `build_tree`, `max_value`。绝对避免拼音命名。
- **常量/宏定义：** 全部大写，单词间用下划线分隔，例如 `MAX_N`, `INF`。

## 2. 结构与排版
- **大括号位置：** 左大括号 `{` 必须紧跟在语句的同一行，不另起一行。
- **空格习惯：** 运算符（`=`, `+`, `-`, `<<`）两侧必须有空格。逗号后面必须有空格。
- **头文件：** 刷算法题时，允许使用 `#include <bits/stdc++.h>`。

## 3. 注释偏好
- 核心逻辑的上方必须有单行注释 `//` 解释原理。
- 尽量不使用 `/* */` 多行注释。
